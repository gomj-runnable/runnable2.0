# 10. 화면설계서 (Screens)

페이지별 화면설계서. 각 페이지는 별도 문서로 분리한다.

- **[10-1. 메인 지도 페이지 (`/`)](10-1-Screen-Map)** — 경로 그리기/비교/탐색/공유 기능이 통합된 메인 작업 화면
- **[10-2. 플러그인 설정 (`/settings`)](10-2-Screen-Settings)** — per-user 플러그인 토글
- **[10-3. 관리자 대시보드 (`/admin`)](10-3-Screen-Admin)** — 카드 분기점 스켈레톤
- **[10-4. 공유 경로 (`/share/[routeId]`)](10-4-Screen-Share)** — 인증 없이 접근 가능한 공유 뷰어

## 화면설계서 규약

모든 화면설계서는 다음 섹션을 동일한 순서로 갖는다.

1. **개요** — 페이지 ID / 라우트 / 파일 / 목적 한 줄
2. **진입 경로** — 어디서 이 페이지로 오는가
3. **화면 레이아웃** — 와이어프레임(이미지) + 영역 설명. 이미지는 사용자가 직접 추가
4. **컴포넌트 구성** — 화면에 등장하는 컴포넌트와 그 책임
5. **인터랙션** — 사용자 액션 → 시스템 반응
6. **데이터 / API** — 호출되는 엔드포인트와 의존 store
7. **권한 / 비로그인 동작** — 인증 필요 여부, 미충족 시 분기
8. **관련 코드** — 핵심 파일 경로
9. **관련 PR / 이슈** — 추적용

## 이미지 정책

- 모든 스크린샷은 `docs/wiki/images/screen-{page-id}-{role}.png` 경로로 추가한다.
- 1차 작성 단계에서는 `<!-- TODO(image): {설명} -->` 주석 + placeholder 박스로 자리를 잡아둔다.
- 캡처 사이즈: 데스크탑 기준 1440×900 권장.
