# 위치 정보 동의 정책

> **[초안]** 본 문서는 서비스 출시 전 법률 전문가 검토 및 위치정보법 준수 점검이 필요한 초안입니다.

**시행일: 미정**
**버전: 0.1 (초안)**

---

## 1. 개요

Runnable은 「위치정보의 보호 및 이용 등에 관한 법률」(이하 위치정보법)에 따라 이용자의 위치 정보를 처리합니다. 위치 정보는 이용자가 명시적으로 동의한 경우에만 수집합니다.

## 2. 위치 정보 수집 목적

| 목적 | 설명 |
|------|------|
| 경로 제작 지원 | 현재 위치를 지도 중심점으로 설정하여 경로 그리기 시작점 제안 |
| 근처 공공시설 표시 | 현재 위치 주변 화장실, 음수대 등 공공시설 정보 제공 |
| 날씨 정보 제공 | 현재 위치 기준 날씨 및 미세먼지 정보 제공 |

## 3. 위치 정보 수집 방법

- **브라우저 Geolocation API**를 통해 수집 (HTML5 표준)
- 이용자가 브라우저의 위치 접근 허용을 승인한 경우에만 작동
- 서버에 저장되지 않으며 클라이언트 측에서만 처리됨

## 4. 동의 흐름

### 4-1. 첫 위치 정보 요청 시

위치 정보가 필요한 기능을 처음 사용할 때 다음 절차를 따릅니다.

```
[앱 내 동의 안내]
   ↓
이용자 확인 ("허용" / "거부")
   ↓
[브라우저 권한 요청]
   ↓
이용자 승인 / 거부
```

**앱 내 동의 안내 문구 (예시):**
> "경로 제작을 시작하려면 현재 위치가 필요합니다. 위치 정보는 서버에 저장되지 않으며 지도 표시 목적으로만 사용됩니다."

### 4-2. 동의 거부 시

- 위치 정보 없이도 서비스 이용 가능 (위치 기반 기능 제외)
- 경로 제작 시 지도를 수동으로 이동하여 시작점 설정 가능
- 날씨, 시설물 정보는 지도 중심 기준으로 표시

### 4-3. 동의 철회

- 브라우저 설정에서 위치 접근 권한 철회 가능
- 서비스 설정 > 권한 관리에서도 철회 가능 (구현 예정)

## 5. 위치 정보 처리 규칙

| 항목 | 처리 방식 |
|------|----------|
| 서버 저장 | **하지 않음** — 클라이언트에서만 처리 |
| 제3자 제공 | 하지 않음 |
| 정밀도 | 브라우저 기본값 (약 10~100m 오차) |
| 수집 시점 | 이용자가 위치 기반 기능 사용 시점에 1회 수집 |
| 백그라운드 수집 | 하지 않음 |

## 6. 위치정보 사업자 신고

위치정보법 제9조에 따라 위치정보사업 신고 의무가 있는지 법률 검토가 필요합니다.

- **검토 필요:** 개인 위치정보 수집·이용 여부에 따른 신고 의무 확인
- **검토 예정:** 서비스 출시 전 법률 자문

## 7. 구현 체크리스트 (개발팀)

위치 정보 동의 흐름 구현 시 확인 사항:

- [ ] 위치 정보 요청 전 앱 내 동의 안내 모달 표시
- [ ] 브라우저 권한 거부 시 대체 UX 제공 (수동 위치 설정)
- [ ] 개인정보처리방침에서 위치 정보 처리 항목 명시적 링크
- [ ] 위치 접근 권한이 없는 상태에서도 핵심 기능 사용 가능 보장
- [ ] 위치 데이터가 네트워크 요청에 포함되지 않는지 확인
