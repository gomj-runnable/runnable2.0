# `server/http` 패키지 안내

H3(Nitro) 요청 처리에서 **API 핸들러가 공통으로 반복하는 전처리**를 모아 둔
횡단 헬퍼 패키지입니다. 인증·인가 요구, 세션 조회, 경로 파라미터 검증처럼 거의
모든 핸들러가 필요로 하는 보일러플레이트를 한곳에서 제공합니다.

도메인 규칙은 담지 않습니다 — 실제 비즈니스 로직은 `server/services`·
`server/repositories` 에 위임하고, 이 패키지는 **요청을 받아 핸들러로 넘기기 전후의
어댑터 역할**만 합니다.

> 이 문서는 패키지의 **역할**만 설명합니다.
> 어떤 파일이 어떤 함수를 제공하는지(파일 세부)는 코드가 유일한 출처이므로,
> 문서가 구현에 종속되지 않도록 여기 적지 않습니다.

## 제공하는 것

- 인증 요구 — 미인증 요청을 차단하고 세션 사용자를 주입
- 관리자 권한 요구 — 권한 미달 요청을 차단
- 세션 조회·요구
- 경로 소유권 확인 (도메인 조회 콜백 주입 기반)
- URL 경로 파라미터 추출·검증

이들은 예외 처리 래퍼(`server/exceptions`)와 합성해 핸들러를 구성하는 방식으로
쓰입니다.

## 도메인에 묶이지 않는 이유

소유권 확인처럼 도메인 데이터가 필요한 기능도 **조회 콜백을 주입받아** 검사만
수행합니다(예: "이 경로의 소유자인가?"는 경로 조회 함수를 인자로 받음).
덕분에 이 패키지는 특정 도메인 레포지토리 구현에 의존하지 않고(타입만 참조),
어떤 도메인 핸들러에서든 재사용됩니다.

## 의존 방향

- **의존하는 것** — `server/security/auth`(세션 인증), `#shared/constants`(권한·역할),
  `server/exceptions`(에러 팩토리), `h3`
- **소비하는 곳** — `server/api/**` 의 이벤트 핸들러
- 도메인 서비스·레포지토리 구현을 직접 import 하지 않습니다(타입 또는 주입 콜백으로만 접근).

## 새 헬퍼를 어디에 둘지 판단하는 기준

`defineEventHandler` 단위에서 **여러 핸들러가 공통으로 반복하는 요청 전처리**
(인증·인가·파라미터 검증·요청 컨텍스트 세팅 등)이면 이 패키지에 둡니다.
특정 도메인에서만 쓰는 로직은 해당 `server/services` 에 두는 편이 응집도가 높습니다.
