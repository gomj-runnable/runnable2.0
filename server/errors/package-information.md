# `server/errors` 패키지 안내

API 핸들러가 **일관된 에러 응답을 만들고, 예상치 못한 예외를 일관된 HTTP
응답으로 변환**하도록 돕는 횡단 패키지입니다. 상태 코드별 에러 생성과 핸들러
예외 래핑을 한곳에서 제공해, 모든 엔드포인트가 같은 형태의 에러를 내도록 합니다.

도메인 규칙은 담지 않습니다 — 무엇이 잘못됐는지(검증·권한·충돌 등)는 호출하는
쪽이 판단하고, 이 패키지는 **그 판단을 표준 HTTP 에러로 표현하는 수단**만 제공합니다.

> 이 문서는 패키지의 **역할**만 설명합니다.
> 어떤 파일이 어떤 함수를 제공하는지(파일 세부)는 코드가 유일한 출처이므로,
> 문서가 구현에 종속되지 않도록(파일 추가·이동·리네임마다 문서를 고치는 의존성을
> 피하도록) 여기 적지 않습니다.

## 제공하는 것

- 상태 코드별 에러 팩토리 — 4xx(요청 측 오류)·5xx(서버 측 오류)를 표준 형태로 생성
- 핸들러 예외 래퍼 — 핸들러에서 던져진 예외를 가로채 일관된 HTTP 응답으로 변환하고
  서버측에 로깅 (의도된 HTTP 에러는 그대로 전파, 검증 오류는 세부 노출 없이 400)
- 단일 공개 진입점(배럴) — 소비처는 개별 파일이 아니라 패키지 한 곳에서 가져옵니다

## 왜 한곳에 모으는가

- **응답 일관성** — 엔드포인트마다 제각각인 에러 형태를 막고, 클라이언트가 항상
  같은 구조의 에러를 받도록 합니다.
- **민감 정보 차단** — 스키마 세부나 내부 스택이 클라이언트로 새지 않도록, 변환
  지점을 한곳으로 좁힙니다.
- **호출부 단순화** — 핸들러는 "무엇이 잘못됐는지"만 표현하면 되고, 응답 변환·
  로깅 같은 공통 처리는 래퍼가 책임집니다.

## 의존 방향

- **의존하는 것** — `h3`(에러 생성 프리미티브)
- **소비하는 곳** — `server/api/**` 핸들러, `server/http`(공통 래퍼 합성),
  `server/services`(도메인 충돌·미존재 표현)
- 도메인 서비스·레포지토리를 직접 import 하지 않습니다.
