# `server/security` 패키지 안내

인증·인가 등 **보안 관심사**를 모아 둔 패키지입니다.
도메인 로직(예: `server/services` 의 경로·시설물 규칙)과 분리된 **인프라 계층**으로,
도메인이 무엇이든 재사용·교체할 수 있도록 도메인에 의존하지 않습니다.

> 이 문서는 패키지의 **역할**만 설명합니다.
> 어떤 파일이 어떤 함수를 제공하는지(파일 세부)는 코드가 유일한 출처이므로,
> 문서가 구현에 종속되지 않도록 여기 적지 않습니다.

## 다루는 것

현재는 better-auth 기반 **세션 인증**(`auth/`)을 책임집니다.
인증 인스턴스 초기화, 요청 세션 조회·검증, 인증 관련 환경 변수 검증이 여기에 속합니다.
HTTP 요청 어댑터나 페이지 가드처럼 이를 *소비*하는 쪽은 다른 패키지가 맡습니다.

## 의존 방향

- **의존하는 것** — `server/config`(실행 환경 모드), `server/database`(better-auth
  Drizzle 어댑터·DB 연결)
- **소비하는 곳** — `server/http`(인증·관리자·세션 래퍼),
  `server/middleware`(페이지 가드), `server/api/auth`(better-auth HTTP 핸들러)
- 도메인 서비스(`server/services`)를 **import 하지 않습니다.**
  보안은 도메인을 몰라야 교체·테스트가 쉽습니다.

## 새 보안 모듈을 어디에 둘지 판단하는 기준

인증·인가·세션·CSRF·비밀 관리처럼 **특정 도메인과 무관하게 앱 전반을 떠받치는
보안 기능**이면 이 패키지에 둡니다.
반대로 특정 도메인(경로·시설물 등) 규칙에 묶인 권한 로직은 그 도메인 서비스에
두는 편이 응집도가 높습니다.
